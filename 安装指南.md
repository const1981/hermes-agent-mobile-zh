# Hermes Android App · 安装指南（中文）

> 一句话结论：**先装「Hermes Android App」这个 APK，不用先装 Termux。**
> App 自带运行环境（内置 proot + 自动下载 Ubuntu），点开就能一键装好 Hermes，全程不需要你打开 Termux 敲命令。

---

## 〇、两种路线，先认清楚

| 路线 | 是什么 | 需要先装 Termux？ | 适合谁 |
|------|--------|------------------|--------|
| **A. App 路线（主推，本文档）** | 安装我们的 APK，App 内部一键装好一切 | ❌ **不需要** | 普通用户，点图标即用 |
| B. 纯 Termux 脚本路线 | 不用 App，直接在 Termux 里跑 `scripts/install-termux.sh` | ✅ 需要 | 只想用命令行、不要 App 壳的人 |

> 仓库里 `scripts/install-termux.sh` 和 `README.md` 旧版描述的是**路线 B**。本文档讲的是**路线 A（APK）**，也是咱们 GitHub Actions 自动构建出来的那个。
> 如果你只想用路线 B，见文末「附：纯 Termux 脚本路线」。

## 一、路线 A：安装 Hermes Android App（推荐）

### 第 1 步：装 APK
1. 从发布页下载 APK：
   - GitHub Actions 构建产物：`https://github.com/const1981/hermes-agent-mobile-zh/actions`（最新一次成功构建的 `hermes-android-app-apk`  artifact，约 30MB）
   - 或仓库 Release（若有）
2. 把 APK 传到手机，点击安装。
3. 首次安装会提示「允许安装未知来源应用」→ 允许。本 APK 为 debug 签名，可直接安装，无需 Google Play。

> 安装只需约 30MB；但首次运行会从网上下载 Ubuntu 运行环境（见第 2 步），请确保手机有 **约 500MB~1GB 空闲存储 + 稳定联网**。

### 第 2 步：首次启动，一键装运行环境
1. 打开 **Hermes Android App**。
2. 首次进入会显示 **Setup 向导**（标题「Setup Hermes Agent」）。
3. 点 **「Begin Setup」**。App 会自动完成以下 5 步（屏幕有进度，约几分钟，需联网）：
   - ① 下载 Ubuntu 24.04 基础系统（rootfs）
   - ② 解压 rootfs 到 App 私有目录
   - ③ 安装 Python / git 等基础包
   - ④ 克隆并安装 Hermes Agent（上游 `nousresearch/hermes-agent`，建 Python venv + pip 装依赖）
   - ⑤ 配置环境
4. 看到 **「Setup complete!」** → 点 **「Go to Dashboard」**。

> 这 5 步**全部在 App 内部完成**，没有 Termux、没有命令行。如果中途失败，点「Retry Setup」重来；多数是网络问题（rootfs 或 PyPI 下载慢）。

### 第 3 步：启动网关
1. 进 Dashboard 后，点 **「启动网关」**（Start Gateway）。
2. 网关跑在手机本机 `127.0.0.1:18789`，App 内置面板直接对话，无需额外浏览器。

### 第 4 步：配置大模型服务商 + API Key（必须做）
App 只负责跑 Agent，调用大模型需要你自己的 API Key。

1. 在 Dashboard 点 **「Configure」（配置）**。
2. App 会打开一个**内置终端**，自动跑 `hermes setup` 向导（在 proot Ubuntu 里）。
3. 按终端提示操作：
   - 选大模型提供商：OpenRouter / 智谱 / DeepSeek / Kimi / 自定义 OpenAI 兼容
   - 粘贴你的 **API Key**（输入不回显，可长按粘贴）
   - 确认默认模型（OpenRouter 默认 `openai/gpt-4o-mini`，可自行改）
   - 可选：填 Telegram Bot Token（用 Telegram 给 Agent 发消息）
4. 配置完终端显示「Configuration complete!」→ 返回。

> ⚠ **OpenRouter 已无免费模型**，请用你自己的**付费 key**（任意 OpenAI 兼容模型均可）。
> 国内智谱/DeepSeek/Kimi 有免费或赠送额度，新手可先拿它们试。
> Key 只存在手机本机 `~/.hermes/.env`，**不会写进 App 代码、不会上传任何服务器**。

### 第 5 步：后台保活（锁屏也能跑）
安卓会杀后台进程，锁屏后 Agent 可能停。按 [docs/后台保活指南.md](docs/后台保活指南.md) 设置：
- 电池优化 → 不优化（Hermes App）
- 开启 Termux/App 唤醒锁
- Android 12+ 关掉「Phantom Process Killer」（需 ADB 一条命令）

## 三、常见问题

**Q：一定要先装 Termux 吗？**
A：**不用。** 路线 A（APK）自带 proot + 自动下载 Ubuntu，完全自给自足。Termux 只是另一条可选路线的前置（见附）。

**Q：安装卡在「Downloading rootfs」？**
A：rootfs 约几百 MB，国内网络可能慢。确保联网稳定，点「Retry Setup」重来；也可挂代理后重试。

**Q：启动网关后没反应 / 对话报错？**
A：多半是 Key 没配或配错。回 Dashboard 点「Configure」重跑 `hermes setup` 核对 Key。

**Q：换模型 / 换 Key 怎么改？**
A：Dashboard → Configure → 重跑 `hermes setup` 向导，或直接改手机里 `~/.hermes/config.yaml` 与 `~/.hermes/.env`。

**Q：iOS 能用吗？**
A：本 App 只做安卓。iOS 沙盒限制无法本地驻留 Hermes 进程。

**Q：发布前想查 APK 有没有把 Key 打进去？**
A：用本仓库的 APK 安全检测技能（`~/.workbuddy/skills/apk-security-scan`，需在 Termux/Linux 跑 aapt2+apktool）扫描 smali 里的 `api_key/token/secret`。正确做法下 Key 只在用户手机本机，不会进 APK。

## 附：纯 Termux 脚本路线（可选，需先装 Termux）

> 这条路线**不用 App**，直接在 Termux 里跑我们写的脚本，Hermes 跑在原生 Termux（不下载 Ubuntu、更轻）。**此路线才需要先装 Termux。**

1. 装 [Termux](https://f-droid.org/zh_Hans/packages/com.termux/)（F-Droid 或 GitHub 版，Play 版较旧）。
2. 首次打开 Termux 执行：
   ```bash
   pkg update && pkg upgrade -y
   termux-setup-storage
   ```
3. 一键安装（效果等同 App 内按钮）：
   ```bash
   bash <(curl -fsSL https://raw.githubusercontent.com/const1981/hermes-agent-mobile-zh/main/scripts/install-termux.sh)
   ```
4. 按提示选服务商、填 Key，装完直接 `hermes` 开聊。
