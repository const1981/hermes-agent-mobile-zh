# 让 Hermes 在安卓后台持续运行（保活指南）

安卓为了省电，会主动杀死后台进程——包括 Termux 和里面跑的 Hermes。如果你看到 Termux 提示
`[Process completed (signal 9) - press Enter]`，说明安卓刚刚把 Hermes 干掉了。

本指南让你一次设置好，之后锁屏也能 24/7 跑。约 2 分钟，一次性。

---

## 第 1 步：开启 Termux 唤醒锁（Wake Lock）

1. 下拉通知栏，找到 **Termux 的通知**
2. 点 **Acquire wakelock（获取唤醒锁）**

作用：阻止安卓把 Termux 进程挂起。只要 Termux 在前台/有唤醒锁，Hermes 就不会被冻住。

> 进阶：在 Termux 里执行 `termux-wake-lock` 也可常驻唤醒（需装 `termux-api` 插件并授权）。

---

## 第 2 步：关闭电池优化（Battery Optimization）

1. 设置 → 电池 → 电池优化（或"不受监视的应用"）
2. 找到 **Termux**
3. 设为 **不优化**（部分机型叫"无限制 / 允许后台运行"）

不同品牌路径不同，见下方"各品牌速查"。

---

## 第 3 步：关闭 Phantom Process Killer（Android 12+ 关键）

Android 12 及以上会**自动杀死后台子进程**（即使父进程活着）。这是 Hermes 被秒杀的头号原因，
必须用 ADB 关掉它。

### 3.1 开启"无线调试"
1. 设置 → 关于手机 → 连续点击"版本号" 7 次，开启**开发者选项**
2. 设置 → 开发者选项 → 开启 **无线调试**
3. 点 **使用配对码配对设备**，记下出现的 **配对端口** 和 **配对码**

### 3.2 在 Termux 里配对 ADB
```bash
pkg install -y android-tools
adb pair localhost:<配对端口> <配对码>
```

### 3.3 连接
```bash
adb connect localhost:<连接端口>
```
（连接端口在"无线调试"主界面查看，与配对端口不同）

### 3.4 关闭 Phantom Process Killer
```bash
adb shell "settings put global settings_enable_monitor_phantom_procs false"
```

### 3.5 验证
```bash
adb shell "settings get global settings_enable_monitor_phantom_procs"
```
返回 `false` 即成功。该设置**重启后依然保留**，只需做一次。

完成后可以关掉"无线调试"，不影响设置。

---

## 第 4 步：电池保护（可选但推荐）

手机长期插电 100% 会鼓包。建议限制充电上限到 80%：

- **三星**：设置 → 电池 → 电池保护 → 最大 80%
- **Google Pixel**：设置 → 电池 → 电池保护 → 开启
- **小米 / Redmi**：设置 → 电池与性能 → 电池保护（部分机型在"省电与电池"里）

---

## 各品牌省电设置速查

国产机额外激进，仅关电池优化往往不够，还需处理"自启动 / 关联启动 / 锁屏清理"：

- **小米 / Redmi（MIUI / HyperOS）**
  - 设置 → 应用设置 → 应用管理 → Termux → 省电策略 → 无限制
  - 安全中心 → 应用管理 → Termux → 自启动 → 允许
  - 安全中心 → 优化加速 → 锁屏清理 → 把 Termux 加入白名单（或关闭锁屏清理）
  - 设置 → 电量和性能 → 神隐模式 → 关闭（或把 Termux 设为无限制）

- **华为 / 荣耀（EMUI / MagicOS）**
  - 设置 → 应用 → 应用启动管理 → Termux → 手动管理 → 允许自启动 / 关联启动 / 后台活动
  - 手机管家 → 清理加速 → 设置（齿轮）→ 受保护应用 → 勾选 Termux
  - 电池 → 更多电池设置 → 高耗电提醒 → 关闭（避免弹窗打断）

- **OPPO / realme（ColorOS）**
  - 设置 → 电池 → 更多设置 → 耗电保护 → Termux → 关闭"后台冻结/检测到异常时自动优化"
  - 设置 → 应用管理 → Termux → 消耗电量和流量 → 允许后台运行 / 允许自启动
  - 手机管家 → 权限隐私 → 自启动管理 → 允许 Termux

- **vivo / iQOO（OriginOS / Funtouch）**
  - 设置 → 电池 → 后台耗电管理 → Termux → 允许后台高耗电
  - 设置 → 应用与权限 → 应用管理 → Termux → 权限 → 自启动 → 开启
  - i 管家 → 应用管理 → 自启动管理 → 允许 Termux

- **三星（One UI）**
  - 设置 → 应用程序 → Termux → 电池 → 不受限制
  - 设置 → 设备维护 → 电池 → 后台使用限制 → 关闭"睡眠应用/深度休眠"
  - 安全文件夹无关；用"应用程序"里的"自动运行"允许 Termux

> 通用排查：若仍被杀，访问 https://dontkillmyapp.com 按具体机型找方案。

---

## 第 5 步：用 tmux 兜底（推荐）

即使做了上面所有设置，SSH 断开或异常退出仍可能丢现场。用 `tmux` 包一层，断开后 Hermes 仍在跑：

```bash
pkg install -y tmux
tmux new -s hermes          # 新建会话并启动 hermes
# Ctrl+B 再按 D  detach 退出，Hermes 继续后台跑
tmux attach -t hermes       # 随时回来看现场
```

---

## 故障排查

- **还是被杀**：部分厂商（小米/华为/三星）有额外激进策略，按上面"各品牌速查"逐项检查；并确认 Termux 已加入"自启动/白名单"。
- **唤醒锁掉了**：Termux 被强制停止后唤醒锁会失效，重新打开 Termux 并再次 `termux-wake-lock`。
- **Phantom 设置被重置**：极少数系统更新后会重置，重跑第 3 步的 `settings put` 即可。
- **日志在哪**：网关日志 `~/.hermes/logs/gateway.log`；Hermes 报错看 `~/.hermes/logs/`。
